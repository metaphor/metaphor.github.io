VirtualBox Network Adapter

Feb 6th, 2013

A network adatper is the way for your virtual machine instances to connect outside world, rather working as a standalone node.

VirtualBox has 4 types of network adapters

VirtualBox has basically 4 types of network adapters, provding comprehensive means to build your network topology.

NAT (Network Address Transform)

With this adpater, the virtual instance can access the outside world by forwarding the request to host, and the host might re-farword the response back to the virtual instance. The forwarding rules could be configured.

The disadvantage is that the outside world knows only the host, rather the viratual instance. This means you cannot access the instance directly.

Bridged

Bridged adapter might be the easiest and most user friendly. It makes the virtual instance a new physical node like in your network environment, separating from the host. It could obtain IP address from DHCP server.

Such ease leads to an inconvience, your instance network configuration changes as long as your host network environment changes. That means each time you need a different IP to connect to the instance.

Internal Network

This adapter usually uses static IP and only allows virtual instance to communicate with other instances generated by the same VirtualBox. This result in that even the host cannot access the instance. Of course, the instance has no Internet access.

Host Only

This is my favorite adpater, it combines advantages both of Bridged and Internal Network. Virtual instances and host can commnunicate with each other without any problem. But the instance cannot connect the Internet, because it does not connect to the network environment of the host.

To configure Host only adapter, you need to set up a host only network first. On Mac OS, goes to Preferences –> Network –> Add(Host only network)

How to choose right adapters

Each adapter has its own limitations, so the best solution is to configure those adapters in a combined way. VirtualBox allows you to configure multiple adpaters within same instances. Also different use cases have their own combinations.

Instances within same host

Sometimes your instances would be created only in the same host, eg, experimental cluster on you laptop. In this case, I would like to combine,

  • NAT
  • Host-only

This combination allows the virtual instances to access Internet. The host and virtual instances can consistently communicate with each other over static IP.

Instances acrossing hosts

In real world, there is no such experiment, if you want to leverage the VirtualBox for your infrastructure, you probly need to start new instances across different hosts. In this case, it requires the commnunication between instances across hosts. The host only adapter might not fit in this case, so I would like to use,

  • Bridged
  • Switch infrastructure

Bridged instance connect to your network infrastructure via switch, working as new physical machine. Basically, this allow virtual instances to access the Internet as well if your network infrastructure allows.
